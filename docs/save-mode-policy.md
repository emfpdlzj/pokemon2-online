# Save Mode Policy

싱글 저장과 멀티 저장은 같은 사용자라도 별도 슬롯 집합으로 관리한다.

## UI 규칙

- 시작 화면의 세이브 메뉴는 `싱글 저장`, `멀티 저장` 탭으로 나뉜다.
- `싱글 저장`
  - 슬롯을 누르면 즉시 해당 진행도를 불러와 게임을 시작한다.
  - 비어 있는 슬롯을 누르면 새 싱글 진행이 시작된다.
- `멀티 저장`
  - 슬롯을 누르면 곧바로 게임을 시작하지 않고, 해당 슬롯을 현재 멀티 슬롯으로 선택한 뒤 방 목록으로 이동한다.
  - 방 메뉴에서도 현재 멀티 슬롯 1~3을 다시 바꿀 수 있다.
  - 선택한 슬롯에 저장이 있으면 방 입장 시 그 진행도를 먼저 복구하고, 서버 스냅샷으로 현재 방 위치를 덮어쓴다.
  - 선택한 슬롯이 비어 있으면 방 입장 시 새 멀티 진행이 시작된다.

## 저장 시점

- 싱글/멀티 모두 기존 `saveGame()` 자동 저장 타이머를 사용한다.
- 멀티에서는 현재 선택한 멀티 슬롯에만 저장한다.
- 방 재접속, 다른 방 재입장, 같은 방 재입장 모두 마지막으로 선택한 멀티 슬롯을 기준으로 이어진다.
- 멀티 저장은 방 입장 전에 슬롯을 고르고, 방 안에서는 그 슬롯에 계속 덮어쓰는 방식으로 단순화한다.

## 슬롯 구분 기준

- 활성 슬롯과 활성 세이브 ID는 `single`, `multi` 별도로 저장한다.
- 따라서 싱글 슬롯 2를 사용 중이어도 멀티 슬롯 1 진행과 충돌하지 않는다.

## 기존 공용 세이브 이전 정책

현재 서버 마이그레이션 이전 데이터 중 사용자 소유자가 없는 세이브는 `legacy-default` 사용자로 백필된다.

운영 정책은 아래 순서를 따른다.

1. 운영 전환 전에 `user_id = 'legacy-default'` 인 세이브를 모두 백업한다.
2. 실제 소유자를 확정할 수 있는 환경만 관리자 스크립트로 특정 사용자에게 일괄 귀속한다.
3. 소유자를 확정할 수 없는 공용 데이터는 첫 로그인 사용자에게 자동 귀속하지 않는다.
4. 소유권이 불명확한 데이터는 읽기 전용 검토 대상이나 별도 이전 작업으로 분리한다.
5. 신규 사용자 구조 전환 이후에는 `legacy-default` 세이브를 운영 기준 정상 데이터로 취급하지 않는다.

자동 귀속을 막는 이유는, 공용 테스트 데이터가 우연히 첫 접속자 계정으로 흡수되면 다른 사용자 세이브와 구분할 근거가 사라지기 때문이다.
