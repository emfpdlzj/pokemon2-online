# Save Sync Policy

세이브는 항상 브라우저 임시본과 서버 저장을 함께 다룬다. 서버가 실패해도 현재 진행도는 즉시 브라우저에 남기고, 서버 동기화는 나중에 다시 시도한다.

## 저장 상태 표시

- 게임 HUD 아래에 저장 상태 바를 항상 표시한다.
- 상태는 아래 값 중 하나를 사용한다.
  - `저장 중`: 최신 진행도를 서버에 쓰는 중
  - `동기화`: 서버 저장 완료
  - `임시 보관`: 서버 저장 실패, 브라우저 임시본만 최신
  - `충돌`: 로컬 임시본과 서버 저장의 최신 시각이 갈린 상태
  - `오류`: 복구 불가능한 입력/환경 오류
- 상태 바에는 마지막 로컬 저장 시각, 마지막 서버 저장 시각, 자동 재시도 안내를 함께 보여준다.

## 실패와 재시도

- `saveGame()`이 호출되면 먼저 로컬 임시본에 저장한다.
- 서버 저장이 실패하면 조용히 버리지 않고 상태 바를 `임시 보관`으로 바꾼다.
- 서버 저장 실패 시 현재 브라우저 임시본을 유지한 채 지수 backoff로 자동 재시도한다.
  - 2초
  - 4초
  - 8초
  - 최대 15초 간격 유지
- 사용자는 상태 바의 `재시도` 버튼으로 즉시 다시 서버 저장을 시도할 수 있다.

## 충돌 우선순위

브라우저 임시본과 서버 저장이 모두 있을 때는 `savedAt` 과 `updatedAt`을 비교한다.

1. 서버 저장이 없으면 로컬 임시본을 사용한다.
2. 로컬 임시본 시간이 더 최신이면 로컬 임시본을 우선 사용한다.
3. 서버 저장 시간이 더 최신이면 서버 저장을 우선 사용한다.
4. 시간이 같으면 서버 저장을 기준으로 본다.

이 규칙은 세이브 슬롯 화면과 실제 불러오기 흐름 모두에 동일하게 적용한다.

## 슬롯 불러오기 동작

- 싱글 저장
  - 서버 저장만 있으면 서버 저장을 불러온다.
  - 로컬 임시본이 더 최신이면 로컬 임시본을 먼저 불러오고, 이후 다시 저장하면 같은 서버 세이브 ID에 덮어쓴다.
- 멀티 저장
  - 방에 입장하기 전에 선택한 슬롯 기준으로 서버 저장과 로컬 임시본을 비교한다.
  - 로컬 임시본이 더 최신이면 해당 진행도로 방에 들어가고, 서버 스냅샷은 방 위치/동기화 정보만 갱신한다.

## 설명 가능한 데이터 유실 범위

- 브라우저 저장소를 삭제하면 로컬 임시본은 사라진다.
- 서버 저장이 아직 끝나지 않은 상태에서 브라우저 저장소까지 지워지면 해당 진행은 복구할 수 없다.
- 반대로 서버 저장이 완료된 뒤에는 다른 기기에서도 마지막 서버 저장 시점 기준으로 복구할 수 있다.
