# 초안 기능명세서
HTML, CSS, JavaScript (그외 필요하다면 더 써도되긴 함)만 사용해서 브라우저에서 바로 실행 가능한 2D 포켓몬풍 데모 게임을 만들어줘.
빌드 도구 없이 정적 파일만으로 동작해야 하며, AWS S3 정적 호스팅에 바로 올릴 수 있는 구조로 작성해줘.

중요:
- 개인 테스트용 데모다.
- 맵 탐험, 이벤트 대화, 스타팅 선택, 소꿉친구와의 대화 기능이 핵심이다.
- 전체 목표는 "게임 시작 → 박사 등장 → 스타팅 선택 → 소꿉친구와 첫 만남 → 마을 이동 → 두 번째 마을 또는 초반 길목까지 이동"이다.
- 실제 포켓몬 정식 게임 전체 구현이 아니라, 초반 체험형 데모 느낌이면 된다.
- 코드는 최대한 단순하고 읽기 쉽게 작성해줘.
- 처음부터 너무 거대한 구조로 만들지 말고, 확장 가능한 선에서 간단하게 만들어줘.

필수 기능:
1. 시작 화면
- 게임 제목 표시
- "새 게임 시작" 버튼
- 버튼 클릭 시 인트로 시작

2. 박사 인트로 이벤트
- 박사가 플레이어에게 세계관을 소개
- 플레이어 이름은 고정값 또는 간단 입력으로 처리 가능
- 스타팅 몬스터 3마리 중 1마리를 선택할 수 있어야 함

3. 스타팅 선택
- 불 / 물 / 풀 계열 느낌의 3마리 중 하나 선택
- 선택 결과를 game state에 저장
- 선택 후 박사 대사와 함께 연구소 밖으로 이동

4. 맵 이동
- 키보드 방향키로 플레이어 이동
- 최소 3개 맵:
  - 집 또는 시작 장소
  - 박사 연구소
  - 첫 번째 마을
  - 초반 길 / 두 번째 마을 입구 중 하나
- 맵 간 출입구 타일을 밟으면 화면 전환
- 벽/장애물 충돌 처리

5. NPC 이벤트
- 고정 대사 NPC 몇 명
- 소꿉친구 NPC 1명
- NPC 근처에서 상호작용 키를 누르면 대화창 표시
- 기본 NPC는 스크립트 대사 사용

6. 소꿉친구 LLM 대화
- API 키는 코드 상단 상수에 직접 넣는 방식으로 구성
- fetch 기반으로 단순하게 구현
- 응답은 1~2문장 정도로 짧게 표시
- 실패 시에는 기본 대사 fallback 출력

7. UI
- 레트로 RPG 느낌
- 하단 또는 중앙에 대화 박스
- 상단 한쪽에 현재 위치 / 선택한 스타팅 표시
- 전체적으로 픽셀풍 느낌이 나면 좋지만 구현은 단순해도 됨

8. 저장
- localStorage로 현재 진행 상태 저장
- 최소 저장 항목:
  - 선택한 스타팅
  - 현재 맵
  - 플레이어 좌표
  - 주요 이벤트 완료 여부

9. 파일 구조
다음처럼 단순한 구조로 만들어줘:
- index.html
- style.css
- game.js
- data/maps.js
- data/dialogues.js
- assets/... (필요시)

10. 개발 원칙
- 함수 분리 명확히
- game state 객체 하나로 주요 상태 관리
- 매직넘버 최소화
- 주석 적당히 포함
- 외부 프레임워크 없이 바닐라 JS 우선
- 처음 실행 시 바로 플레이 가능해야 함

LLM 연동 관련:
- 소꿉친구의 말투는 활발하고 경쟁심 있는 소꿉친구 느낌
- 답변은 짧고 게임 세계관 분위기를 유지
- 현대 밈, AI라고 말하기, 메타 발언 금지
- 현재 초반 마을 범위 밖 이야기는 하지 않도록 프롬프트 제한


최종적으로는 각 파일 코드를 전부 작성해줘.


추가 요구사항:

- canvas 없이도 가능하면 DOM 기반보다 canvas 기반 2D 렌더링이 더 자연스러우니, 가능하면 canvas로 구현해줘.
- 타일 기반 맵처럼 보이게 해줘.
- 플레이어는 사각형 또는 간단 스프라이트 플레이스홀더여도 괜찮다.
- NPC도 간단한 컬러 박스/사각형으로 시작해도 된다.
- 중요한 건 그래픽 퀄리티보다 게임 흐름이다.

상호작용 방식:
- 방향키: 이동
- Enter 또는 Space: NPC와 대화 / 대화 진행
- 대화 중 선택지가 있으면 버튼 클릭 또는 숫자 선택 가능

맵 설계:
- 시작 집
- 연구소
- 첫 마을
- 2번째 지역 입구
이 4구역 정도면 충분하다.

이벤트 흐름:
1. 시작 화면
2. 집 밖으로 나감
3. 박사 연구소로 유도
4. 스타팅 선택
5. 연구소 밖에서 소꿉친구 등장
6. 첫 마을 이동
7. 길목 또는 다음 지역 입구 도착
이 흐름이 자연스럽게 이어지게 구성해줘.

대화 시스템:
- 텍스트 한 줄씩 출력
- 다음 버튼 또는 Enter로 넘기기
- 상태에 따라 대사가 달라지게 해줘

LLM 채팅 UI:
- 소꿉친구 대화창 안에 "자유 대화" 버튼 추가
- 누르면 입력창과 전송 버튼 노출
- 응답을 말풍선처럼 보여줘
- 네트워크 오류 시 "나중에 다시 말 걸어줘!" 같은 기본 문구 출력

코드는 반드시 실제 실행 가능한 형태로 작성해줘.



LLM API 연동부는 수정하기 쉽게 작성해줘.

요구사항:
- game.js 상단에 아래 상수들을 분리해줘
  - API_KEY
  - API_URL
  - MODEL_NAME
  - RIVAL_SYSTEM_PROMPT
- fetch 요청 함수는 별도 함수로 분리
- 응답 파싱도 별도 함수로 분리
- 추후 내가 다른 LLM API로 바꾸기 쉽게 최대한 의존성을 줄여줘
- 실패 시 fallback 대사를 반환하는 구조로 만들어줘
