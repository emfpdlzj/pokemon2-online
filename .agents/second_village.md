# 두 번째 마을

기존 프로젝트에 "2번째 마을" 구역을 추가해줘.
Kiro가 터지지않게 한 번에 나누어서 조금씩 작업할 것. 

중요:
- 현재 게임은 초반 포켓몬풍 데모이며, 전체 완성보다 자연스러운 이동 흐름과 분위기가 중요하다.
- 기존 구조를 최대한 유지하면서 확장해줘.
- 코드가 너무 복잡해지지 않게 해줘.
- 이번 작업의 목표는 "첫 번째 마을에서 길을 지나 두 번째 마을로 도착하는 체험"을 만드는 것이다.

추가할 내용:

1. 새로운 맵 추가
- 두 번째 마을 맵을 새로 만들어줘.
- 기존 첫 번째 마을에서 연결되는 출구를 통해 진입 가능해야 함. 이동하는데 1분 정도 걸려야함. 
- 맵 크기는 너무 크지 않게, 탐색 가능한 소형 마을 형태로 만들어줘.
- 타일 기반 구조를 유지해줘.
- 최소 요소:
  - 집 2~3채
  - 길
  - 나무 또는 장식물
  - 포켓몬센터 
  - 상점
  - NPC 몇 명
  - 한 곳은 아직 막혀 있거나 "공사중" 느낌으로 막아도 됨

2. 맵 연결
- 첫 번째 마을 → 길(또는 중간 맵) → 두 번째 마을
- 기존에 길 맵이 없다면 간단한 중간 길 맵도 추가해줘.
- 출입구 타일을 밟으면 맵 전환되게 해줘.
- 맵 전환 시 플레이어 시작 좌표를 자연스럽게 설정해줘.

3. 두 번째 마을 분위기
- 초반 마을답게 조용하고 소박한 느낌
- 너무 복잡한 구조는 말고, 간단하지만 "새 지역에 왔다"는 느낌이 나게 해줘.
- 현재 게임의 레트로 RPG 분위기를 유지해줘.
- 이름은 나팔꽃마을

4. NPC 추가
- 두 번째 마을에 NPC 2~4명 추가
- 기본 NPC는 고정 대사 사용
- 대사 예시 방향:
  - 다른 마을에서 왔냐고 묻는 주민
  - 체육관 또는 다음 지역을 암시하는 주민
  - 몬스터와의 여행에 대해 말하는 주민
- 대사는 짧고 게임풍으로 작성해줘.

5. 이벤트 추가
- 두 번째 마을에 처음 도착했을 때 1회성 도착 메시지 출력
  예:
  "새로운 마을에 도착했다!"
- 처음 도착 시 한 번만 실행되도록 game state에 이벤트 플래그 저장
- 필요하면 마을 입구 근처에 간단한 안내 이벤트도 추가

6. HUD / 위치 표시
- 현재 위치 표시가 있다면 두 번째 마을 이름도 반영해줘.
- 이름은 코드에서 쉽게 수정할 수 있도록 상수 또는 맵 데이터로 분리해줘.

7. 저장
- localStorage 저장 대상에 두 번째 마을 진행 상태도 포함해줘.
- 현재 맵과 좌표 복원 시 두 번째 마을에서도 정상 동작해야 함.

8. 코드 구조
- 가능하면 맵 데이터는 data/maps.js 같은 데이터 파일에 추가
- NPC 및 대사는 dialogues 또는 별도 데이터로 분리
- 하드코딩을 최소화하고, 맵 추가가 쉬운 구조로 정리해줘

9. 구현 우선순위
우선순위는 아래 순서로 해줘:
1) 맵 연결
2) 두 번째 마을 맵 생성
3) NPC 배치
4) 도착 이벤트
5) 저장 반영

10. 결과물 정리
코드만 주지 말고 아래도 같이 정리해줘:
- 어떤 파일을 수정했는지
- 두 번째 마을 진입 조건
- 추가된 NPC 목록
- 내가 마을 이름/대사를 수정해야 할 위치

추가 스타일 가이드:
- 지금 프로젝트의 하트골드풍 초반 마을 느낌을 유지해줘.
- 그래픽보다 흐름과 분위기 우선
- 너무 많은 기능을 한 번에 넣지 말고, "실제로 플레이 가능한 새 마을"을 목표로 해줘.
