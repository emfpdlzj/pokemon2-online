# 턴제 전투 시스템
기존 프로젝트에 "포켓몬 스타일의 초간단 턴제 전투 시스템"을 추가해줘.

중요:
- 실제 포켓몬 시스템처럼 복잡하게 만들지 말고, 2시간 내 구현 가능한 수준으로 단순화해줘.
- UI는 레트로 포켓몬 느낌을 참고하되 최소한으로 구성해줘.
- 코드 구조는 기존 game.js와 자연스럽게 연결되게 해줘.

전투 시작 조건:
- 특정 위치에서 이벤트 발생 시 전투 화면으로 전환
- 또는 NPC와 상호작용 시 전투 시작

---

전투 시스템 요구사항:

1. 전투 화면 구성
- 플레이어 몬스터 (왼쪽 아래)
- 적 몬스터 (오른쪽 위)
- 하단에 대화창
- 버튼 2~3개:
  - 공격
  - 도망 (선택사항)
  - 스킬 (선택사항)

---

2. 몬스터 데이터 구조
간단한 구조로 정의:

- name
- hp
- maxHp
- attack

예:
{
  name: "Flameling",
  hp: 30,
  maxHp: 30,
  attack: 5
}

---

3. 턴제 로직
- 플레이어 행동 → 적 행동 순서
- 공격 시:
  - hp 감소
  - 로그 출력

예:
"Flameling의 공격! 5 데미지!"

---

4. 승패 조건
- 적 hp <= 0 → 승리
- 플레이어 hp <= 0 → 패배

---

5. 전투 흐름
1. 전투 시작 메시지
2. 플레이어 행동 선택
3. 플레이어 공격
4. 적 공격
5. 반복
6. 승리 또는 패배

---

6. UI
- canvas 또는 DOM 기반 둘 다 가능
- 대화 텍스트 한 줄씩 출력
- 버튼 클릭으로 진행

---

7. 애니메이션 (간단하게)
- 공격 시 살짝 흔들림 또는 위치 이동 정도만

---

8. 상태 관리
- battleState 객체로 관리
- 기존 gameState와 분리

---

9. 전투 종료 후
- 기존 맵으로 복귀
- 승리 시 메시지 출력

---

10. 코드 요구사항
- battle.js로 분리 가능하면 분리
- 함수 단위로 깔끔하게 작성
- 기존 구조 깨지지 않게

---

추가 제한:
- 타입 상성 없음
- 스킬 1개만 사용
- 아이템 없음
- 레벨 시스템 없음

목표:
"포켓몬 느낌이 나는 최소한의 전투 체험"


추가 요구사항:

- HP를 숫자뿐 아니라 간단한 HP 바로 시각화
- 공격 시 로그가 순차적으로 출력되도록 (타이핑 효과 또는 딜레이)
- 플레이어와 적의 몬스터 이미지 표시
- 전투 시작 시 "야생의 ○○가 나타났다!" 메시지 출력
